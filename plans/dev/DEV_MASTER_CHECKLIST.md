# Dev Master Checklist

## Mục tiêu

Theo dõi delivery, QA, release, ops và readiness cho toàn bộ `WS2`.

## Checklist

- [x] Tạo hệ thống plan trong repo.
- [x] Tạo status dashboard.
- [ ] Chốt môi trường local/test/staging/prod.
- [ ] Chốt owner cho từng broker.
- [ ] Chốt dependency với Middleware Hub.
- [ ] Chốt Kafka topics, ACL và retention.
- [ ] Chốt secrets strategy.
- [ ] Chốt QA/UAT checklist.
- [ ] Chốt release và rollback checklist.
- [ ] Chốt runbook incident.

## Dependencies

- `RISKS_AND_DEPENDENCIES.md`
- `dev/ENVIRONMENT_READINESS_PLAN.md`
- `dev/RELEASE_READINESS_PLAN.md`
- `dev/SECURITY_COMPLIANCE_PLAN.md`

## Exit Criteria

- [ ] Mỗi broker có delivery plan riêng.
- [ ] Mỗi release có readiness checklist.
- [ ] Mỗi blocker có owner.
